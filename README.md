# Indian Stocks — Financial Performance Dashboard

A password-protected financial dashboard hosted on GitHub Pages.
Each company's data is fetched by a Python script, stored as a JSON file, and displayed in the browser.

---

## Folder Structure

```
dashboard/
├── index.html          ← main dashboard (open this in browser, or visit GitHub Pages URL)
├── fetch_data.py       ← Python script to scrape and save data
├── requirements.txt    ← Python packages needed
├── data/
│   └── LAURUSLABS.json ← one JSON file per company (auto-generated by fetch_data.py)
└── README.md           ← this file
```

---

## Step 1 — Install Python and dependencies (first time only)

You need Python 3.8 or newer.

Open Terminal and run:

```bash
pip install -r requirements.txt
```

This installs: `requests`, `beautifulsoup4`, `lxml`

---

## Step 2 — Fetch data for a company

Run this command from inside the `dashboard/` folder:

```bash
python fetch_data.py LAURUSLABS
```

Replace `LAURUSLABS` with any NSE ticker symbol (e.g., `HDFCBANK`, `INFY`, `TITAN`).

The script will:
1. Fetch annual data from screener.in (consolidated view)
2. Calculate derived metrics (NPM%, CAGR, RoA, D/E, Div%)
3. Fetch shareholding history from trendlyne.com
4. Fetch year-end stock prices from trendlyne.com
5. Fetch quarterly P&L from screener.in
6. Save everything to `data/LAURUSLABS.json`

If a data source is unavailable, that cell is marked red — the script never crashes.

---

## Step 3 — View the dashboard

**Locally:** Double-click `index.html` to open it in your browser.

**Online (GitHub Pages):** Visit `https://snehanimmagadda5.github.io/stockperformance/`

Enter the password when prompted, then type or select a ticker and click **Load Data**.

---

## Updating data each quarter

Each quarter, repeat Step 2 for every company you track:

```bash
python fetch_data.py LAURUSLABS
python fetch_data.py HDFCBANK
python fetch_data.py INFY
```

Then push the updated JSON files to GitHub (see Deployment below).

---

## Adding a new company

Just run the fetch script with the new ticker:

```bash
python fetch_data.py TITAN
```

The JSON file is created automatically. On GitHub Pages, the dashboard will list it in the dropdown after you push and the page refreshes.

---

## Changing the dashboard password

1. Open `index.html` in a text editor.
2. Find this line near the top of the `<script>` section:
   ```js
   const DASHBOARD_PASSWORD = "stocks2025";
   ```
3. Replace `stocks2025` with your new password.
4. Save and push to GitHub.

The password is hashed in the browser using SHA-256 — the plaintext is never stored.

## Deploying to GitHub Pages

The repository is already created: `https://github.com/snehanimmagadda5/stockperformance`

**First-time setup:**


The dashboard updates automatically within ~1 minute of pushing.

## Color coding

| Color | Meaning |

| Green (#C6EFCE) | Directly pulled from a data source |

## Data sources

| Source | What it provides |
|--------|----------------|
| screener.in (consolidated) | Sales, NP, EPS, Balance Sheet, Ratios (annual + quarterly) |
| trendlyne.com | Shareholding history, historical stock prices |
| moneycontrol.com | Fallback if trendlyne is unavailable |

## Troubleshooting

**"No data file found for TICKER"** — Run `python fetch_data.py TICKER` first.

**Script fetches but values are all red** — The source website layout may have changed. Check that the ticker exists on screener.in by visiting `https://www.screener.in/company/TICKER/consolidated/` in your browser.

**Password prompt keeps rejecting** — Make sure you're using the correct password. It's case-sensitive.

**GitHub Pages not loading** — Check that Pages is enabled (Settings → Pages) and that `index.html` is in the root of the `main` branch.
